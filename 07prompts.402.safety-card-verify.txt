핵심요구사항) 다음의 기능을 supabase를 연동하여 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1. PIN 검증 시나리오
                1-1) 사용자가 입력한 PIN을 supabase safety_cards 테이블의 데이터와 비교하여 검증할 것
                        - 참고 스키마: /Users/gharam12/Desktop/team/code-camp_SAFE_Meals/docs/schema.md
                        - 테이블: safety_cards
                        - 검증 방식
                            - 현재 로그인한 사용자의 user_id로 safety_cards 테이블 조회
                            - 조회된 pin_code와 입력된 pin_code 일치 여부 확인
                        - 검증 필드
                            - pin_code: VARCHAR(4) (필수, 정확히 4자리, 숫자만 허용)
                        - 유효성 검증
                            - 빈 값 검증: pin_code가 비어있으면 에러
                            - 길이 검증: pin_code가 정확히 4자리가 아니면 에러
                            - 형식 검증: 숫자만 허용, 그 외 문자는 에러

                1-2) PIN 검증 성공 이후 로직을 완성할 것.
                        - 알림메시지: 없음
                        - 상태 변경: 잠금 해제 상태로 변경하여 안전카드 내용(message_ko, message_en 등) 표시

                1-3) PIN 검증 실패 시 로직을 완성할 것.
                        - 에러 메시지: "PIN 번호가 일치하지 않습니다."
                        - 상태 유지: 잠금 상태 유지

핵심요구사항) 테스트 파일을 다음 경로에 생성할 것.
            - 테스트 파일 경로: /src/features/profile/components/safety-card/tests/verify.spec.ts
            - 테스트 시나리오
                1) PIN 입력 폼이 렌더링되는지 확인
                2) 4자리 PIN 입력 후 제출 시 검증 성공 및 안전카드 내용 표시
                3) 잘못된 PIN 입력 시 에러 메시지 표시
                4) 4자리가 아닌 PIN 입력 시 유효성 검증 실패
                5) 숫자가 아닌 문자 입력 시 유효성 검증 실패
