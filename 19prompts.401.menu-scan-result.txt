1) 테스트 제외 라이브러리

2) 테스트 조건
   - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
   - 페이지가 완전히 로드된 후 테스트할 것.
       - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
       - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

3) 테스트 데이터 조건
   3-1) 데이터
       - useMenuGrouping 훅의 실제 데이터를 사용할 것.
       - Mock 데이터를 사용하지 말 것.

   3-2) 성공 시나리오
       - 메뉴 그룹핑 완료 후 결과 표시 검증

   3-3) 실패 시나리오
       - 메뉴 항목이 없을 때 빈 화면 처리 검증

핵심요구사항) useMenuGrouping 훅의 결과를 result-area 영역에 바인딩하여 표시할 것.

핵심요구사항) 다음 데이터 바인딩을 step-by-step으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
                1) 바인딩할 데이터
                    - useMenuGrouping 훅의 MenuGroupingResult

                2) 데이터 바인딩 상세내용
                    - 메뉴명: menuItems[].name
                    - 가격: menuItems[].price
                    - 신뢰도: menuItems[].confidence (백분율로 표시)
                    - 전체 항목 수: totalItems
                    - 평균 신뢰도: averageConfidence

                3) UI 상태 처리
                    - 로딩 중: isProcessing === true
                        - 로딩 스피너 또는 스켈레톤 표시
                    - 데이터 없음: menuItems.length === 0
                        - "메뉴를 찾을 수 없습니다" 메시지 표시
                    - 에러: error !== null
                        - 에러 메시지 표시 및 재시도 버튼

                4) CSS 처리
                    - 메뉴 리스트: 스크롤 가능한 영역
                    - 각 메뉴 항목: 카드 형태
                    - 신뢰도에 따른 색상 표시
                        - 높음 (0.8 이상): 녹색
                        - 보통 (0.5 ~ 0.8): 주황색
                        - 낮음 (0.5 미만): 회색 (반투명)

                5) 테스트 시나리오
                    - 메뉴 항목이 정상적으로 표시되는지 검증
                    - 로딩 상태에서 데이터 로드 상태로 전환되는지 검증
                    - 빈 결과 처리가 정상적으로 표시되는지 검증
                    - 신뢰도에 따른 UI 변화 검증

핵심요구사항) 사용자 인터랙션 기능을 구현할 것.
                1) 메뉴 항목 선택
                    - 체크박스로 개별 선택
                    - 전체 선택/해제 버튼

                2) 선택된 메뉴 내보내기
                    - JSON 형식 다운로드
                    - 클립보드 복사

                3) 재분석 버튼
                    - 다시 OCR 실행
                    - 새로운 이미지 업로드
