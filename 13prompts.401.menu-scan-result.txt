핵심요구사항) Edge Function 분석 결과를 화면에 바인딩할 것.

1) 바인딩할 데이터 소스
   - 데이터 저장소: useState 또는 Router state (이전 페이지에서 전달받음)
   - 데이터 구조
        ```typescript
        interface AnalysisResult {
          overall_status: 'SAFE' | 'CAUTION' | 'DANGER';
          detected_ingredients: string[];
          warnings: Array<{
            ingredient: string;
            allergen: string;
            severity: 'HIGH' | 'MEDIUM' | 'LOW';
          }>;
          message_ko: string;
          message_en: string;
        }
        ```

2) 데이터 바인딩 상세내용
   2-1) overall_status별 UI 분기 (Tailwind CSS 활용)
        - SAFE
            - 배경색: bg-[var(--color-success-light)] 또는 bg-green-50
            - 텍스트 색상: text-[var(--color-success)] 또는 text-green-700
            - 아이콘: ✓ 체크 아이콘 (초록색)
            - 메시지: message_ko 또는 message_en (language 설정 따름)
        - CAUTION
            - 배경색: bg-[var(--color-warning-light)] 또는 bg-yellow-50
            - 텍스트 색상: text-[var(--color-warning)] 또는 text-yellow-700
            - 아이콘: ⚠ 경고 아이콘 (주황색)
            - 메시지: message_ko 또는 message_en
            - 경고 목록: warnings 배열을 리스트로 표시
        - DANGER
            - 배경색: bg-[var(--color-error-light)] 또는 bg-red-50
            - 텍스트 색상: text-[var(--color-error)] 또는 text-red-700
            - 아이콘: ✕ 또는 ⛔ 위험 아이콘 (빨강색)
            - 메시지: message_ko 또는 message_en
            - 경고 목록: warnings 배열을 리스트로 표시 (강조)

   2-2) 감지된 재료 목록 표시 (Tailwind CSS 활용)
        - detected_ingredients 배열을 태그 또는 리스트로 표시
        - 재료명: 각 재료를 작은 태그로 표시 (px-3 py-1 rounded-full bg-gray-100 text-sm)
        - 레이아웃: 가로로 나열하되, 줄바꿈 허용 (flex flex-wrap gap-2)

   2-3) 경고 목록 표시 (CAUTION, DANGER인 경우) (Tailwind CSS 활용)
        - warnings 배열을 카드 또는 리스트로 표시 (space-y-2)
        - 각 경고 항목: (p-4 rounded-lg border)
            - 재료명 (ingredient): font-semibold
            - 알레르기 유발 물질 (allergen): text-sm text-gray-600
            - 심각도 (severity): HIGH > MEDIUM > LOW 순으로 시각적 강조
        - 심각도별 색상 구분 (Tailwind 클래스)
            - HIGH: bg-red-100 border-red-300 text-red-800
            - MEDIUM: bg-orange-100 border-orange-300 text-orange-800
            - LOW: bg-yellow-100 border-yellow-300 text-yellow-800

   2-4) 다국어 메시지 처리
        - language 설정에 따라 message_ko 또는 message_en 표시
        - language가 'ko'면 message_ko, 'en'이면 message_en
        - 언어 설정은 useLanguageStore 또는 props로 전달받음

3) 재촬영 버튼
   - 버튼 클릭 시 /scan 페이지로 돌아가기
   - 버튼 텍스트: "다시 촬영하기" (ko) / "Retake Photo" (en)
   - 버튼 위치: 하단 고정

4) 데이터 없음 처리
   - analysis 데이터가 없는 경우
        - 에러 메시지: "분석 결과를 불러올 수 없습니다."
        - "다시 촬영하기" 버튼 표시
        - 자동으로 /scan 페이지로 리다이렉트 (선택사항)

5) Tailwind CSS 처리
   - globals.css의 CSS 변수를 bg-[var(--color-name)] 형태로 활용
   - 상태별 배경색: bg-gradient-to-br 또는 단색
   - 카드 레이아웃: shadow-md rounded-xl p-6
   - 반응형: 모바일 우선 (sm:, md:, lg: 활용)
