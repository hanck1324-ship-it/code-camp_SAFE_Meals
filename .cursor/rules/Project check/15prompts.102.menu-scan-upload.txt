아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc
            - @03-ui.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/app/menu-scan/components/upload-section/index.tsx

==============================================

조건-스타일) Tailwind CSS 유틸리티 클래스만 사용할 것.
            - CSS 모듈 파일 생성 금지
            - 인라인 스타일 금지
            - global.css에 정의된 CSS 변수는 Tailwind에서 활용 가능

조건-공통목록) Tailwind CSS 디자인 시스템 활용
            - 색상: Tailwind 기본 컬러 팔레트 또는 커스텀 테마 색상
            - 타이포그래피: Tailwind 텍스트 유틸리티 (text-sm, text-base, font-medium 등)
            - 간격: Tailwind 스페이싱 시스템 (p-4, m-2, gap-3 등)

==============================================

핵심요구사항) 모바일 최적화된 이미지 업로드 UI를 구현할 것.

핵심요구사항) 다음 기능을 step-by-step으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
                1) camera-button 영역
                    - 카메라 촬영 버튼
                    - 갤러리 선택 버튼
                    - 각 버튼은 명확한 아이콘과 레이블 표시

                2) preview-area 영역
                    - 이미지 선택 전: 안내 문구 표시 ("메뉴판을 촬영하거나 선택해주세요")
                    - 이미지 선택 후: 선택된 이미지 미리보기
                    - 이미지 삭제/재선택 버튼

                3) process-button 영역
                    - OCR 분석 시작 버튼
                    - 이미지 선택 전에는 비활성화 상태
                    - 이미지 선택 후 활성화

핵심요구사항) 파일 입력 처리 조건을 구현할 것.
                - 허용 파일 형식: image/jpeg, image/png, image/webp
                - 최대 파일 크기: 10MB
                - 크기 초과 시 에러 메시지 표시
