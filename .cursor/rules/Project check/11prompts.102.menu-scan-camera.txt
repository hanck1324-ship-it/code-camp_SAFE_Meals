핵심요구사항) 메뉴판 사진 촬영을 위한 카메라 UI를 구현할 것.

1) 카메라 접근 기능
   - 사용자의 디바이스 카메라에 접근
   - 카메라 권한 요청 및 처리
   - 전면/후면 카메라 전환 기능 (선택사항)

2) 사진 촬영 기능
   - 촬영 버튼 클릭 시 사진 캡처
   - 촬영된 이미지를 컴포넌트 내부 상태로만 보관
   - **중요금지사항** 서버로 전송하지 말 것 (다음 단계에서 처리)

3) 미리보기 기능
   - 촬영된 사진을 화면에 표시
   - 재촬영 버튼 제공
   - 사진 삭제 시 다시 카메라 화면으로 전환

4) 분석하기 버튼
   - 사진이 촬영된 상태에서만 활성화
   - 버튼 클릭 시 상위 컴포넌트로 이미지 데이터 전달 (onCapture prop)
   - 버튼 상태: 기본(비활성), 활성, 로딩 상태 구분

5) 접근성 및 사용성
   - 버튼에 명확한 레이블 제공
   - 카메라 권한 거부 시 안내 메시지 표시
   - 로딩 상태 표시 (카메라 초기화 중)

6) 컴포넌트 Props 인터페이스
   - onCapture: (imageData: string | File) => void
   - onClose: () => void
   - language: 'ko' | 'en' (다국어 지원)

핵심요구사항) 현재 존재하는 CameraScreen 컴포넌트를 리팩토링하여 위 기능을 완성할 것.
            - 기존 구조를 유지하되, 실제 카메라 촬영 로직을 구현할 것.
            - navigator.mediaDevices.getUserMedia API 활용
            - HTML5 Canvas 또는 File API 사용하여 이미지 데이터 추출

핵심요구사항) Tailwind CSS를 활용하여 반응형으로 구현할 것.
            - 모바일 우선 디자인
            - 카메라 뷰포트는 전체 화면 활용 (h-screen, w-full)
            - 버튼은 하단 고정 배치 (fixed bottom-0)
            - globals.css의 CSS 변수를 bg-[var(--color-name)] 형태로 활용
