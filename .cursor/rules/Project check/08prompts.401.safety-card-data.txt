       - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

3) 테스트 API 조건

       - 실제 Supabase 데이터를 사용할 것.

   3-2) 성공 시나리오
       - PIN 검증 성공 후 safety_cards 데이터를 로드하여 표시

   3-3) 실패 시나리오
       - 데이터 로드 실패 또는 빈 데이터 처리

핵심요구사항) PIN 검증 성공 후 안전카드 데이터를 로드하여 표시하는 기능을 React Query + Supabase로 구현할 것.
            1. 데이터 로드 로직
                1-1) 다음의 조건으로 Supabase safety_cards 테이블에서 데이터를 조회할 것
                        - 참고 스키마: /docs/schema.md
                        - 테이블: safety_cards
                        - 조회 방식
                            - React Query의 useQuery 훅 사용
                            - Supabase Client로 현재 로그인한 사용자의 user_id로 조회
                            - 조회 쿼리:
                                supabase
                                  .from('safety_cards')
                                  .select('*')
                                  .eq('user_id', user.id)
                                  .single()
                        - 조회 필드
                            - message_ko: TEXT (한국어 메시지)
                            - message_en: TEXT (영어 메시지)
                            - message_ja: TEXT (일본어 메시지)
                            - message_zh: TEXT (중국어 메시지)
                            - message_local: TEXT (현지어 메시지)

                1-2) 언어 설정에 따라 표시할 메시지를 선택할 것
                        - 언어 설정 소스: useLanguageStore (Zustand)
                            - 경로: @/commons/stores/useLanguageStore
                            - 사용 방법: const { language } = useLanguageStore()
                        - 언어별 필드 매핑
                            - language === 'ko' → message_ko 표시
                            - language === 'en' → message_en 표시
                            - language === 'ja' → message_ja 표시
                            - language === 'zh' → message_zh 표시
                            - 기타 언어 → message_ko를 기본값으로 표시
                        - Fallback 처리
                            - 선택된 언어의 메시지가 없으면 message_ko 사용
                            - message_ko도 없으면 빈 문자열 표시

            2. 상태 관리
                2-1) React Query로 다음 상태를 관리할 것
                        - isLoading: 데이터 로딩 중
                        - isError: 데이터 로드 실패
                        - data: 조회된 safety_cards 데이터

                2-2) 잠금 상태와 데이터 로드 상태를 분리할 것
                        - PIN 검증 전: 잠금 상태 (PIN 입력 폼 표시)
                        - PIN 검증 성공 후: 잠금 해제 상태
                            - 데이터 로딩 중: 로딩 인디케이터 표시
                            - 데이터 로드 성공: 메시지 표시
                            - 데이터 로드 실패: 에러 메시지 표시

            3. UI 표시 로직
                3-1) PIN 검증 성공 후 다음 순서로 UI를 업데이트할 것
                        - 1단계: 잠금 해제 상태로 변경
                        - 2단계: 데이터 로딩 시작 (로딩 인디케이터 표시)
                        - 3단계: 데이터 로드 완료 (메시지 표시)

                3-2) 표시할 메시지
                        - 선택된 언어에 맞는 message 필드 표시
                        - 줄바꿈 처리: white-space: pre-wrap 적용
                        - 텍스트 정렬: 가운데 정렬

                3-3) 에러 처리
                        - 데이터 없음: "안전카드 정보가 없습니다." 표시
                        - 로드 실패: "데이터를 불러올 수 없습니다. 다시 시도해주세요." 표시

                3) 언어 변경 시 해당 언어의 메시지로 변경 확인
