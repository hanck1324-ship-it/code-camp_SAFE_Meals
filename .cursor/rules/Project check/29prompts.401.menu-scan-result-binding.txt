       - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
       - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

3) 테스트 데이터 조건
   3-1) 데이터
        - 실제 분석 결과 데이터를 사용할 것.
        - Mock 데이터를 사용하지 말 것.

   3-2) 성공 시나리오
        - analysis 데이터가 존재하는 경우
        - overall_status별 UI 분기 검증

   3-3) 실패 시나리오
        - analysis 데이터가 없는 경우

핵심요구사항) 결과 페이지 컴포넌트를 수정하여 실제 데이터를 바인딩할 것.

1) 하드코딩된 menuItems 제거
   - 현재 하드코딩된 목 데이터 (bibimbap, kimchiJjigae 등) 제거

2) Context에서 분석 결과 가져오기
   - import { useAnalyzeResult } from '@/features/scan/context/analyze-result-context'
   - const { analysisResult, clearAnalysisResult } = useAnalyzeResult()

3) 데이터 없음 처리
   - analysisResult가 null인 경우:
       - 에러 메시지: "분석 결과를 불러올 수 없습니다." (ko) / "Unable to load analysis results." (en)
       - "다시 촬영하기" 버튼 표시
       - data-testid="error-message"

4) overall_status별 UI 분기 (Tailwind CSS 활용)
   - SAFE
       - 배경색: bg-green-50
       - 텍스트: text-green-700
       - 아이콘: CheckCircle (초록색)
       - 메시지: message_ko 또는 message_en
       - data-testid="overall-status-SAFE"
   - CAUTION
       - 배경색: bg-yellow-50
       - 텍스트: text-yellow-700
       - 아이콘: AlertTriangle (주황색)
       - 메시지: message_ko 또는 message_en
       - 경고 목록 표시
       - data-testid="overall-status-CAUTION"
   - DANGER
       - 배경색: bg-red-50
       - 텍스트: text-red-700
       - 아이콘: AlertCircle (빨강색)
       - 메시지: message_ko 또는 message_en
       - 경고 목록 강조 표시
       - data-testid="overall-status-DANGER"

5) 감지된 재료 목록 표시
   - detected_ingredients 배열을 태그로 표시
   - 스타일: px-3 py-1 rounded-full bg-gray-100 text-sm
   - 레이아웃: flex flex-wrap gap-2
   - data-testid="ingredient-tag"

6) 경고 목록 표시 (CAUTION, DANGER인 경우)
   - warnings 배열을 카드로 표시
   - 각 경고: p-4 rounded-lg border space-y-2
       - ingredient: font-semibold
       - allergen: text-sm text-gray-600
       - severity별 색상:
           - HIGH: bg-red-100 border-red-300 text-red-800
           - MEDIUM: bg-orange-100 border-orange-300 text-orange-800
           - LOW: bg-yellow-100 border-yellow-300 text-yellow-800
   - data-testid="warning-item"

7) 메뉴 항목 리스트 표시 (results 배열)
   - results?.map() 사용
   - 각 메뉴: original_name, translated_name, description, safety_status, reason, ingredients
   - safety_status별 아이콘 및 배지 (기존 getSafetyIcon, getSafetyBadge 함수 재사용)
   - data-testid="menu-item"

8) 다국어 메시지 처리
   - language === 'ko' ? message_ko : message_en
   - useLanguageStore에서 language 가져오기

9) 재촬영 버튼
   - 클릭 시: clearAnalysisResult() 호출 후 router.push('/scan')
   - 텍스트: "다시 촬영하기" (ko) / "Retake Photo" (en)
   - data-testid="retake-button"

10) data-testid 추가
    - 전체: "scan-result-screen"
    - 메시지: "status-message"
    - 재료 태그: "ingredient-tag"
    - 경고 항목: "warning-item"
    - 메뉴 항목: "menu-item"
    - 재촬영 버튼: "retake-button"
    - 에러 메시지: "error-message"
