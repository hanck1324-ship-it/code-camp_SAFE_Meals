            - 비슷한 토큰 선택하여 사용(하드코딩 금지)

==============================================

핵심요구사항) OCR/분석된 메뉴를 카드 형태 리스트로 표시할 것.
            1) 카드 레이아웃 구성
                - 메뉴명 (menuName): 카드 상단에 제목으로 표시
                - 간단 설명 (description): 메뉴명 아래 본문으로 표시
                - 안전 등급 (safetyLevel): 카드 우측 상단에 배지 형태로 표시
                - 위험 근거 (riskReasons): 카드 하단 또는 토글 영역에 표시

            2) 리스트 레이아웃
                - 카드들을 세로 방향으로 나열
                - 카드 간 적절한 간격 유지
                - 스크롤 가능한 리스트 영역

핵심요구사항) 안전 등급 배지를 색상 + 아이콘으로 시각화할 것.
            1) 안전 등급별 디자인
                - SAFE: 초록색 배경 + 체크 아이콘 또는 "안전" 텍스트
                - CAUTION: 노란색/주황색 배경 + 경고 아이콘 또는 "주의" 텍스트
                - WARNING: 빨간색 배경 + 위험 아이콘 또는 "위험" 텍스트

            2) 배지 위치
                - 카드 우측 상단 또는 하단에 명확히 표시
                - 텍스트와 겹치지 않도록 배치

            3) 배지 스타일
                - 둥근 모서리 (border-radius)
                - 충분한 padding으로 가독성 확보
                - 아이콘과 텍스트가 함께 표시될 경우 적절한 간격

핵심요구사항) 위험 근거를 인라인 텍스트 또는 토글 UI로 노출할 것.
            1) 위험 근거 표시 방식
                - 카드 내부에 인라인 텍스트로 표시 (기본)
                - 또는 (i) 아이콘 클릭 시 펼쳐지는 토글 UI

            2) 위험 근거 내용
                - riskReasons 배열의 각 항목을 리스트 형태로 표시
                - 예시: "액젓 포함 가능", "소고기 육수 포함"
                - 근거가 없으면 표시하지 않음

            3) 스타일
                - 짧은 근거 문구로 명확하게 표시
                - 팝업이 아닌 인라인 또는 토글 형태로 구현
                - 카드 내부에서 자연스럽게 노출

핵심요구사항) 완성된 페이지는 다음의 구조를 유지할 것.
            1) 페이지 구성
                - 상단: 페이지 제목 또는 헤더 (선택)
                - 중앙: 메뉴 분석 결과 카드 리스트
                - 하단: 추가 액션 버튼 (선택)

            2) 반응형 디자인 (선택)
                - 모바일: 카드 너비 100%
                - 태블릿/데스크탑: 적절한 최대 너비 설정

            3) 빈 상태 처리
                - 분석 결과가 없을 경우 "분석된 메뉴가 없습니다" 메시지 표시
