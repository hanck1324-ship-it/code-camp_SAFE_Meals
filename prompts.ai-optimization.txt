==============================================
AI 호출 최적화 프롬프트 가이드
==============================================

[목적]
AI 호출 시 토큰 절약 및 응답 속도 개선을 위한
두 가지 핵심 최적화 원칙을 적용한다.

==============================================

## 원칙 1: 입력 축소 (토큰 200↓)

적용 범위: 모든 AI 호출
부작용: 없음

1) 핵심만 전달
   - 불필요한 설명/예시/반복 제거
   - 200 토큰 이하로 프롬프트 유지

2) 구조화된 입력
   - 글머리 기호로 요점만 나열
   - 이미 알고 있는 컨텍스트 재전달 금지

3) 적용 예시
   ❌ "사용자가 버튼을 클릭하면 모달이 열리고
       모달 안에 폼이 있어서 데이터를 입력하면..."

   ✅ "버튼 클릭 → 모달 열림 → 폼 입력 → 저장"

==============================================

## 원칙 2: 출력 최소화 + 스트리밍

적용 범위: 첫 토큰 UX가 중요한 모든 호출
효과: 체감 응답 속도 직결

1) 출력 형식 명시
   - "코드만 출력"
   - "설명 없이"
   - "JSON으로만"
   - "3줄 이내"

2) 스트리밍 활성화
   - stream: true 옵션 사용
   - 첫 토큰 0.5초 이내 표시 목표

3) 적용 예시
   ❌ "React 컴포넌트 만들어줘"
   → 긴 설명 + 코드 + 사용법 모두 출력

   ✅ "React 컴포넌트 코드만 (설명 생략)"
   → 코드만 즉시 스트리밍 시작

==============================================

## 프롬프트 작성 템플릿

[작업명]
- 입력: [필수 데이터만]
- 출력: [형식 명시]
- 제한: [길이/설명 생략 등]

예시:
---
로그인 구현
- 입력: email, password
- 출력: TypeScript 코드만
- 제한: 설명 생략, 주석 최소화
---
